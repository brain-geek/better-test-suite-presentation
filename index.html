Why is testing so important?

  1. Testing provides negative feedback
  2. People expect the software to be reliable
  3. Feedback from testing improves quality
  4. Humans make mistakes all the time, we need to catch them

Why do automated testing?

  1. Less manual work - someone does the testing anyway!
  2. Automation is more attentive
  3. Faster feedback

Law of Demeter

The guideline was proposed at Northeastern University towards the end of 1987, and can be succinctly summarized in each of the following ways:

  1. Each unit should have only limited knowledge about other units: only units "closely" related to the current unit.
  2. Each unit should only talk to its friends; don't talk to strangers.
  3. Only talk to your immediate friends.

Architecture summary:

  3. Learn from the both sides

Change something small in the project

Testing is hard.

So, helping members of your team with tests is not bad. Adding few selectors, renaming some inputs isn't hard, but will boost the performance of the automated testing team.

Improving testability improves project quality.

  4. Environments must be as similar as possible
